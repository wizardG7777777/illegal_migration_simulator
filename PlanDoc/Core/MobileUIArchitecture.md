# 手机界面UI架构设计

本文档定义《去美国》游戏的手机界面UI架构，将游戏内容包装为一部智能手机的操作系统，通过各类"APP"承载不同的事件类型，营造沉浸式现代移民体验。

**核心设计理念**：
- **沉浸式模拟**：玩家扮演角色操作手机，所有游戏行为通过手机APP完成
- **信息层级隔离**：不同类型的事件通过不同APP隔离，符合手机使用直觉
- **情绪映射**：手机状态（电量、信号、时间）与角色状态映射，增强代入感
- **现代叙事**：通过APP内容（招聘软件、社交软件、导航等）讲述移民故事

### 2.2 系统状态栏映射

```typescript
interface SystemStatusBar {
  // 时间显示 - 映射回合数
  time: {
    display: string;        // "9:41 AM" / "11:30 PM"
    turnInfo: string;       // 场景1显示"第12天"，场景3显示"第3月第5天"
  };
  
  // 信号强度 - 映射社交属性
  signal: {
    bars: 0 | 1 | 2 | 3 | 4;  // 0-4格信号
    status: 'no_signal' | 'weak' | 'good' | 'excellent';
    tooltip: string;          // "社交值低，人脉网络薄弱"
  };
  
  // 网络类型 - 映射当前场景/状态
  network: {
    type: '2G' | '3G' | '4G' | '5G' | 'WiFi' | 'Roaming' | 'No SIM';
    description: string;
  };
  
  // 电量 - 映射健康+心理综合状态
  battery: {
    percentage: number;       // 0-100
    status: 'charging' | 'normal' | 'low' | 'critical';
    color: string;            // 绿色/黄色/红色
  };
  
  // 通知图标
  notifications: {
    hasUnread: boolean;
    count: number;
    urgent: boolean;          // 紧急通知（终结态警告）
  };
}
```

**状态映射表**：

| 系统状态 | 游戏映射 | 显示规则 |
|---------|---------|---------|
| **时间** | 回合进度 | 场景1=天数(早6点-晚12点)，场景2=倒计时，场景3=月份+日期 |
| **信号** | 社交属性 | 0格=0-2社交，1格=3-5，2格=6-8，3格=9-12，4格=13-20 |
| **网络** | 场景/身份 | 场景1=4G/5G，场景2=Roaming漫游，场景3=WiFi/不稳定网络 |
| **电量** | 健康+心理 | (健康+心理)/2，充电=恢复类道具生效中 |
| **通知** | 随机事件 | 红点数字=待处理事件数，感叹号=紧急事件 |
