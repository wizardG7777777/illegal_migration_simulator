## 一、已有系统回顾：三场景压力塑造

### 1.1 核心设计理念

三个场景采用**递进式压力模型**，从"慢性焦虑"到"急性生存"再到"慢性结构性压迫"，形成完整的情绪曲线。

```
场景1（国内）          场景2（跨境）           场景3（美国）
   ↓                       ↓                       ↓
慢性焦虑 ──────────────→ 急性生存 ──────────────→ 慢性压迫
（温水煮青蛙）          （刀尖舔血）            （笼子困兽）
```

### 1.2 各场景压力机制详解

#### 🏭 场景1：国内准备 —— 【就业冰河期】

| 压力维度 | 具体机制 | 玩家体验 |
|---------|---------|---------|
| **经济压力** | 打工收入递减、汇率波动、物价上涨 | "越拖越穷"的焦虑感 |
| **时间压力** | 30回合后触发"放弃"结局选项 | 被迫做决策的紧迫感 |
| **机会成本** | 三条离境路线门槛不同（走线/旅游签/学生签） | 选择困难与后悔 |
| **社交剥离** | 与家人朋友关系疏远事件 | 渐行渐远的孤独感 |

**关键设计**：
- 无直接死亡威胁，但"慢性失血"让每个决策都沉重
- 打工是唯一的现金来源，但重复性劳动消耗身心健康
- **核心矛盾**：留下（稳定但压抑）vs 离开（未知但"有希望"）

---

#### 🌴 场景2：跨境穿越 —— 【法律真空】

| 压力维度 | 具体机制 | 玩家体验 |
|---------|---------|---------|
| **生存压力** | 每回合强制触发负面事件（6种随机） | "每一步都可能死"的恐惧 |
| **资源枯竭** | 雨林中现金/物资快速消耗 | 无助的绝望感 |
| **方向迷失** | 进度条显示0-5步，但每一步都是赌命 | 看不到尽头的煎熬 |
| **人性考验** | 向导涨价、同伴背叛、道德抉择 | "人吃人"的丛林法则 |

**关键设计**：
- **只能前进，无退路** —— 切断玩家的逃避心理
- 高体魄/风险意识/生存能力是"门票"，不达标即死亡
- 8种死亡结局，每种都有详细死亡描述（倒在某棵树下的无名尸体）

**阶段1负面事件示例**：
```
丛林劫匪 → 风险意识判定 → 失败=枪杀死亡
毒蛇猛兽 → 体魄判定 → 失败=毒发死亡  
危险小径 → 风险意识判定 → 失败=摔死
沼泽陷阱 → 生存能力判定 → 失败=溺死
毒虫袭击 → 体魄判定 → 失败=感染死亡
可疑食物 → 智力判定 → 失败=中毒死亡
```

---

#### 🗽 场景3：美国生存 —— 【搜捕升级】+【大通胀时代】

| 压力维度 | 具体机制 | 玩家体验 |
|---------|---------|---------|
| **现金流断裂** | 每月固定扣除生活成本（$400+$住宿+$出行） | "永远不够花"的恐慌 |
| **身份焦虑** | 无证件打工收入低、被抓风险高 | 随时可能失去一切的恐惧 |
| **结构性压迫** | 通胀叠加、执法压力Debuff累积 | 系统性的无力感 |
| **希望陷阱** | 三条绿卡路径看似可达但门槛极高 | "差点就能留下"的煎熬 |

**关键设计**：
- **道具驱动生活成本**：住宿（收容所→合租→公寓→House）和出行（公交→电动车→车）由道具决定
- **通胀不可逆**：经济型Debuff永久叠加，除非触发特殊事件
- **入狱即终局**：被捕后无保释、无大赦、无遣返流程，长期拘留或狱中终老
- **二元结局**：只能获得绿卡或被捕，禁止转往第三国（避免"逃离"轻松感）

**移民执法压力系统**：
```
强度1（轻微关注）→ 突击检查+5%，打工难度+5
强度2（搜捕升级）→ 突击检查+15%，打工难度+10
强度3（风声鹤唳）→ 现金消耗+10%，庇护暂停
强度4（全城搜捕）→ 现金消耗+20%，无法打工
强度5（红色警戒）→ 突击检查+50%，现金消耗+30%
```

## 二、新事件系统：NPC互动机制

### 2.1 设计目标

在原有"事件驱动"基础上，引入**NPC关系网**，通过对话逐步解锁信息、资源和事件，增强叙事深度和重复可玩性。

### 2.2 核心机制

#### 🎭 NPC发现与解锁流程

```
第一次接触 → 添加通讯录 → 对话互动 → 好感度/信任度积累 → 解锁事件/信息/道具
    ↓
特定条件触发对话（地点/时间/前置事件）
    ↓
对话选择影响关系走向（友好/中立/敌对）
    ↓
关系等级解锁不同互动选项
```

#### 📱 NPC互动场景分布

| 场景 | NPC数量 | 主要互动方式 | 关键NPC示例 |
|------|---------|-------------|------------|
| 场景1 | 5-8人 | 微信/陌陌/线下偶遇 | 蛇头老王、移民群群主、高利贷老板 |
| 场景2 | 3-5人 | 面对面（危险环境） | 墨西哥向导胡安、走私客阿米尔、其他偷渡者 |
| 场景3 | 8-12人 | 微信/WhatsApp/线下 | 教会志愿者、律师、同乡会、黑市中介 |

### 2.3 NPC关系等级设计

每个NPC设置5级关系，通过对话选择提升或降低：

| 等级 | 状态 | 解锁内容 |
|------|------|---------|
| Lv.0 | 陌生人 | 基础信息 |
| Lv.1 | 认识 | 更多背景故事、小道消息 |
| Lv.2 | 熟人 | 低价购买道具、特殊打工渠道 |
| Lv.3 | 朋友 | 借款、担保、关键事件链解锁 |
| Lv.4 | 挚友 | 独特结局、稀有道具、隐藏剧情 |

### 2.4 对话系统设计

#### 对话触发方式

1. **主动发起**：玩家从通讯录选择NPC发起对话
2. **被动接收**：NPC主动发来消息（任务/求助/警告）
3. **场景触发**：到达特定地点自动触发对话
4. **事件关联**：完成某事件后NPC发来反馈

#### 对话选择影响

```
NPC: "最近风声紧，有个活儿接不接？"

选项A: "什么活儿？多少钱？" → 关系+0，解锁打工事件
选项B: "太危险了，算了。" → 关系-1，NPC暂时不联系你
选项C: "先说说看，我再决定。" → 关系+1，了解更多信息后再选择
```

### 2.5 NPC与事件链整合

**示例：庇护申请事件链（场景3）**

```
触发条件：通过NPC"张律师"对话解锁
    ↓
Step 1: 联系张律师（Lv.1+）
    ↓
对话内容：张律师询问基本情况，给出建议
    ↓
Step 2: 准备材料（5回合后解锁）
    ↓
对话内容：张律师指导准备虚假材料，需要选择是否说谎
    ↓
Step 3: 提交申请（再5回合后解锁）
    ↓
对话内容：张律师告知审理需要1-2年，期间不能离境
    ↓
结局分支：
  - 运气好（小概率）：庇护通过 → 通关
  - 被发现说谎：直接遣返
  - 等待中被抓：入狱即终局
```

### 3.2 像素风格规范

#### 视觉参考

- **分辨率模拟**：320×640（iPhone SE尺寸）的像素化呈现

- **字体**：像素点阵字体，中文使用"汉仪像素体"或类似风格
- **动画**：2-4帧简单动画，强调"复古数字感"

### 3.3 状态栏与游戏状态映射

```
手机状态栏元素 → 游戏状态映射

📶 信号强度  →  社交属性（信号越差=越孤立）
🔋 电量      →  健康+心理（综合状态）
⏰ 时间      →  回合进度（早晨/下午/晚上/深夜）
📅 日期      →  总回合数 / 30 = 月份
🌐 网络状态  →  场景2=无信号，部分APP禁用
```

## 四、界面改造：星露谷物语风格场景

### 4.1 设计理念

在保留手机界面作为**主交互层**的同时，引入**像素场景**作为视觉背景，让玩家"看到"主角所处的环境。

**核心原则**：
1. **手机界面始终是操作核心**
2. **像素场景提供氛围和视觉反馈**
3. **避免复杂的场景交互**，场景主要是"展示"
4. **星露谷风格**：45度俯视角、2-3头身角色、明快但略带忧郁的配色

### 4.2 场景分类与展示时机

#### 🏠 居所场景

**触发时机**：
- 回合开始时（早晨醒来）
- 选择"休息"后（夜晚）
- 事件触发时（如房东催租）

**居所状态随剧情变化**：

| 阶段 | 居所类型 | 场景视觉 | 心理暗示 |
|------|---------|---------|---------|
| 场景1-开始 | 出租屋单间 | 狭小但整洁，有书架和电脑 | 还有"正常人"的生活 |
| 场景1-中期 | 群租房床位 | 拥挤混乱，上下铺，噪杂 | 生活质量下降 |
| 场景3-初期 | 家庭旅馆 | 狭小的临时房间，行李箱敞开 | 漂泊感，不稳定 |
| 场景3-中期 | 合租公寓 | 相对整洁，有基本家具 | 暂时的"家"的感觉 |
| 场景3-后期 | 收容所 | 大通铺，简陋，压抑 | 底层生存的真实 |

### 4.3 社交互动设计：手机为主，现实为辅

#### 核心原则

> **"现代人活在手机里"** —— 主角在美国的社交主要通过手机完成，现实环境刻意保持疏离感。

#### 手机社交（主要方式）

**微信/WhatsApp对话**：
- 与NPC的文字聊天
- 群聊中的信息获取
- 朋友圈的动态展示

**设计要点**：
- 对话有"输入中..."提示，模拟真实聊天
- 消息可以有"已读"标记，增加心理压力
- 语音消息使用波形动画（不播放真实音频）

```
│  │      对方输入中     │    │
│  └────────────────────┘    │
└─────────────────────────────┘
```

#### 现实社交（辅助方式）

**刻意减少现实对话**，只在以下情况出现：

1. **场景2（跨境）**：面对面交流（危险环境下的本能互动）
2. **重要剧情节点**：如第一次见律师、被捕时的审讯
3. **工作场景**：简短指令性对话（工头、顾客）

**设计意图**：
- 场景3的"美国生存"强调**孤独感**和**边缘人身份**
- 手机社交创造"连接感"的幻觉，但现实中仍是陌生人
- 现实社交的稀少让每一次面对面都显得重要

### 4.4 星露谷风格实现要点

#### 美术规范

| 元素 | 规格 | 参考 |
|------|------|------|
| 角色 | 2-3头身，16×16或24×24像素 | 星露谷村民 |
| 场景 | 45度俯视角，16×16瓷砖 | 星露谷地图 |
| 配色 | 明快但带灰度，避免过于饱和 | 星露谷 seasons |
| 动画 | 4方向行走，简单待机动画 | 星露谷角色动作 |

#### 技术实现建议

1. **使用Tiled Map Editor**制作场景地图
2. **导出JSON**由游戏引擎加载
3. **角色使用Sprite Sheet**管理动画帧
4. **场景与手机界面分层**：
   - 底层：像素场景（静态或简单动画）
   - 上层：手机界面（主要交互）
