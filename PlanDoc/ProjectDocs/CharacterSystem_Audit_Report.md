#### 2. 属性成长数值检查

#### 3. 终结态窗口计算

**规范要求**: 
- 濒死: 体魄决定喘息窗口（0-3回合）
- 崩溃: 智力决定窗口（1-3回合）

**检查结果**: 
- `TerminalStates.md`中的计算公式与`AGENTS.md`完全一致
- 濒死倒计时: `clamp(floor((体魄 + 6) / 7), 0, 3)`
  - 体魄0-1 → 0回合（立即死亡）
  - 体魄2-8 → 1-2回合
  - 体魄9-15 → 2-3回合
  - 体魄16-20 → 3回合
- 崩溃窗口: `clamp(3 - floor(智力 / 10), 1, 3)`
  - 智力0-9 → 3回合
  - 智力10-19 → 2回合
  - 智力20 → 1回合

---

#### 问题2: 场景3崩溃状态行动点处理不一致

**位置**: `Events_Act3_USA.md` 第1497行

**修复前**:
```markdown
| 效果 | 本回合行动点归零，无法工作，现金收入归零 |
```

**修复后**:
```markdown
| 效果 | 本回合行动点上限-2（难以集中注意力），无法执行工作事件，现金收入归零 |
```

**修复理由**: 
- `TerminalStates.md`规定濒死状态"每回合额外扣除1点行动点"（累加方式）
- 崩溃状态应使用相同的累加逻辑，而非直接归零
- 采用"行动点上限-2"更符合"冷静克制的旁观者视角"的叙事基调
