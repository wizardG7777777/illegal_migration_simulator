# 《去美国》里程碑验收清单

> 本文档定义各开发阶段的验收标准，用于阶段性评审和发布决策。

---

## MVP 验收标准（场景1可玩）

**目标日期**：第 4 周末  
**核心目标**：玩家可以创建角色，在场景1进行完整的游戏循环，体验打工、学习、随机事件，并可以通过不同方式离开场景1（进入场景2或直接到场景3）。

### 功能完整性

#### 角色系统 ✅/⬜
- [ ] **角色创建**：支持预设角色和自定义描述创建
- [ ] **六大属性**：体魄、智力、英语、社交、风险意识、生存能力（范围0-20）
- [ ] **资源系统**：
  - [ ] 身体健康度（0-100，归零进入濒死）
  - [ ] 心理健康度（0-100，归零进入崩溃）
  - [ ] 现金（场景1人民币/场景2、3美元，归零进入匮乏）
  - [ ] 行动点（每回合恢复，限制行动次数）
- [ ] **终结态系统**：
  - [ ] 濒死（DYING）：体魄决定喘息窗口（0-3回合）
  - [ ] 崩溃（BREAKDOWN）：智力决定窗口（1-3回合）
  - [ ] 匮乏（DESTITUTION）：无倒计时，持续扣血扣心理

#### 回合制循环 ✅/⬜
- [ ] **回合开始**：恢复行动点，应用环境Debuff
- [ ] **随机事件判定**：按权重抽取并触发
- [ ] **玩家行动阶段**：显示可用固定事件，执行选择
- [ ] **回合结束**：检查终结态，更新冷却计时

#### 事件系统 ✅/⬜
- [ ] **随机事件**：回合开始自动触发，支持条件、冷却、次数限制
- [ ] **固定事件**：玩家主动选择，支持可重复/一次性配置
- [ ] **事件链**：线性或分支结构，节点依赖
- [ ] **物品槽位**：标签匹配系统，支持强制/可选槽位

#### 物品系统 ✅/⬜
- [ ] **消耗型道具**：可堆叠，一次性使用
- [ ] **常驻型道具**：通过槽位匹配，场景切换时清空
- [ ] **道具栏管理**：两个独立道具栏，容量限制
- [ ] **全局书籍池**：跨场景保留，10本书唯一

#### 场景系统 ✅/⬜
- [ ] **场景1完整**：国内准备阶段所有事件
- [ ] **环境Debuff**：【就业冰河期】打工收入下降，就业困难
- [ ] **场景切换**：支持进入场景2（走线）或场景3（直飞）

#### 存档系统 ✅/⬜
- [ ] **存档功能**：手动存档（5个槽位）
- [ ] **自动保存**：回合结束时自动保存
- [ ] **读档功能**：从存档恢复完整游戏状态
- [ ] **数据完整性**：存档包含角色、场景、物品、事件状态

### 数值平衡

#### 场景1时长 ✅/⬜
- [ ] **预期时长**：25回合左右完成准备
- [ ] **软上限**：50回合上限（过长惩罚）
- [ ] **放弃结局**：30回合后触发"放弃"事件

#### 经济系统 ✅/⬜
- [ ] **起始资金**：2000人民币
- [ ] **打工收入**：合理收入区间（50-150元/次）
- [ ] **物价平衡**：道具价格和效果成正比
- [ ] **Debuff压力**：【就业冰河期】压力适中，体现经济下行感

#### 属性影响 ✅/⬜
- [ ] **属性检定**：影响事件成功率和解锁选项
- [ ] **成长速度**：合理成长曲线，不过快或过慢
- [ ] **属性联动**：属性间协同效果（如高体魄+高生存）

### 体验要求

#### 性能 ✅/⬜
- [ ] **加载时间**：首屏加载 < 3秒
- [ ] **操作响应**：点击到反馈 < 100ms
- [ ] **内存占用**：游戏过程中无显著内存泄漏

#### 兼容性 ✅/⬜
- [ ] **浏览器支持**：Chrome、Firefox、Safari、Edge 最新2个版本
- [ ] **移动端**：基本可玩，布局自适应
- [ ] **离线可用**：支持 Service Worker 离线缓存

#### 稳定性 ✅/⬜
- [ ] **无崩溃**：连续游玩1小时不崩溃
- [ ] **存档可靠**：存档数据完整，读档后状态正确
- [ ] **数据边界**：极端数值下游戏仍能正常进行

## Alpha 验收标准（三场景完整流程）

**目标日期**：第 8 周末  
**核心目标**：三场景完整流程可玩，玩家可以从场景1出发，经过场景2（可选），在场景3生存并达成至少5种不同结局。

### 内容完整性

#### 三场景所有事件 ✅/⬜
- [ ] **场景1**：所有固定事件、随机事件、事件链
- [ ] **场景2**：
  - [ ] 跨境路线事件（向导、漏洞、攀爬）
  - [ ] 环境威胁随机事件（暴雨、野兽、迷路）
  - [ ] 人际事件（同伴加入/离开/背叛）
- [ ] **场景3**：
  - [ ] 打工类事件（外卖、Uber、黑工等）
  - [ ] 身份获取事件链（庇护、婚姻、人才等）
  - [ ] 随机事件（突击检查、通胀、社区事件）

#### 场景切换 ✅/⬜
- [ ] **场景1→2**：走线路径（低门槛，高死亡率）
- [ ] **场景1→3**：直飞路径（签证/学生签，高门槛）
- [ ] **场景2→3**：多种穿越方式（向导安全可重试，其他危险不可重试）
- [ ] **切换惩罚**：清空常驻道具栏，给予对应启动物资

#### 5种结局 ✅/⬜
- [ ] **死亡结局**：濒死/崩溃/环境致死（至少3种）
- [ ] **通关结局**：获得合法身份（至少1种，如庇护）
- [ ] **特殊结局**：遣返/放弃/黑户（至少1种）

### 系统完整性

#### 生活成本系统 ✅/⬜
- [ ] **住宿道具**：收容所/合租/公寓/House，决定住宿成本
- [ ] **出行道具**：公交/电动车/二手车/特斯拉，决定出行成本
- [ ] **食品成本**：固定基础成本
- [ ] **月度扣除**：每月自动扣除（食品+住宿+出行）×通胀率

#### 移民执法压力 ✅/⬜
- [ ] **政策压力事件**：特朗普政策公告事件
- [ ] **Debuff强度**：1-5级，效果累加
- [ ] **影响范围**：检查概率、打工难度、心理消耗、现金消耗
- [ ] **持续时间**：固定回合数后自然衰减

#### 同伴系统 ✅/⬜
- [ ] **同伴获取**：场景2获得同伴
- [ ] **同伴状态**：健康/受伤/分离/死亡
- [ ] **信任度**：影响同伴行为
- [ ] **技能加成**：向导/翻译/医疗等关键技能

#### 死亡机制 ✅/⬜
- [ ] **场景2死亡**：随机环境致命事件
- [ ] **场景3死亡**：终结态倒计时归零
- [ ] **场景过渡死亡**：穿越失败特定描述
- [ ] **死亡结局文案**：每种死因有独特描述

## 验收流程

### MVP验收
1. 自测：开发者对照清单逐项测试
2. 内测：邀请1-2名玩家试玩
3. 评审：确认是否进入Alpha阶段

### Alpha验收
1. 完整流程测试：从场景1玩到场景3结局
2. 多种路径测试：尝试不同场景切换路径
3. 结局测试：触发至少5种不同结局
4. 评审：确认是否进入Beta阶段

### Beta验收
1. 全结局测试：验证24种结局都可达成
2. 数值平衡测试：多位玩家测试反馈
3. Bug修复验证：确认所有严重Bug已修复
4. 评审：确认是否进入Release阶段

### Release验收
1. 部署验证：正式环境部署测试
2. 性能测试：各项性能指标达标
3. 最终评审：确认可对外发布
