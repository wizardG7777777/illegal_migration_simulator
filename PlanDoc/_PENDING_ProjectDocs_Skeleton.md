## 文件 1: ProjectDocs/README.md

```markdown
# ProjectDocs - 项目文档

本文档目录包含《去美国》项目管理相关文档，用于跟踪开发进度、
资源管理和团队协作。

## 适用原则

本项目采用轻量级 Markdown 文档管理，不引入复杂工具：

- **独立/小团队开发**: 简化流程，聚焦内容
- **Git 版本控制**: 所有文档纳入版本管理
- **定期更新**: 每轮开发周期结束后更新文档

## 目录结构

```
ProjectDocs/
├── README.md              # 本文档
├── TaskBoard.md           # 任务看板（当前开发任务）
├── Milestone_Checklist.md # 里程碑验收标准
├── Roadmap.md             # 开发路线图
├── Asset_Inventory.md     # 资源清单
├── Tech_Decisions.md      # 技术决策记录
├── Testing_Plan.md        # 测试计划
└── Playtest_Log.md        # 试玩反馈记录
```

## 使用指南

| 文档 | 更新频率 | 主要读者 |
|-----|---------|---------|
| TaskBoard.md | 每日 | 开发者 |
| Milestone_Checklist.md | 每里程碑 | 开发者/测试者 |
| Roadmap.md | 每周 | 全体 |
| Asset_Inventory.md | 按需 | 开发者 |
```

```markdown
# 里程碑验收清单

## MVP 验收标准（场景1可玩）

### 功能完整性

- [ ] **角色创建**
  - [ ] 可创建随机角色
  - [ ] 可自定义角色（如支持）
  - [ ] 初始属性/资源正确

- [ ] **回合制循环**
  - [ ] 回合开始触发随机事件
  - [ ] 玩家可执行主动事件
  - [ ] 回合结束正确结算
  - [ ] 月度结算（如场景1有）

- [ ] **事件系统**
  - [ ] 所有29个场景1事件可触发
  - [ ] 事件条件判定正确
  - [ ] 选项效果正确应用
  - [ ] 物品槽位匹配工作正常

- [ ] **终结态系统**
  - [ ] 濒死状态可触发
  - [ ] 崩溃状态可触发
  - [ ] 匮乏状态可触发
  - [ ] 救援/恢复路径可用

- [ ] **存档系统**
  - [ ] 可手动存档
  - [ ] 可自动存档
  - [ ] 可读取存档
  - [ ] 存档导入/导出功能正常

### 数值平衡

- [ ] **场景1时长**
  - [ ] 自动模拟10局
  - [ ] 平均回合数在25-40之间
  - [ ] 极端情况下不超过50回合

- [ ] **经济系统**
  - [ ] 打工收入能覆盖基本消耗
  - [ ] 无法无限积累财富
  - [ ] 不同打工选择有显著差异

- [ ] **属性影响**
  - [ ] 高属性有明显优势
  - [ ] 低属性开局仍有通关可能
  - [ ] 属性提升有价值

### 体验要求

- [ ] **性能**
  - [ ] 首屏加载 < 3秒
  - [ ] 事件切换无卡顿
  - [ ] 存档操作响应迅速

- [ ] **兼容性**
  - [ ] 桌面端主流浏览器正常
  - [ ] 移动端可正常游玩
  - [ ] 不同分辨率适配

- [ ] **稳定性**
  - [ ] 无阻断性BUG
  - [ ] 无数据丢失问题
  - [ ] 异常情况有错误提示
```
