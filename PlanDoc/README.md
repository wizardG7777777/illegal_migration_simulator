## 各模块说明

### Core — 核心系统架构

#### 角色系统：状态机驱动的生存实体

角色系统是整个游戏的**状态核心**，所有其他系统（事件、道具、场景）都围绕角色状态运转。技术上采用**集中式状态管理**——角色的六大属性、四项资源、终结态、成长记录全部聚合在一个 Character 对象中，避免状态分散导致的同步问题。

**关键设计决策**：

1. **属性与资源分离**：属性（体魄/智力/英语/社交/风险意识/生存能力）是长期能力值，范围 0-20，影响事件检定；资源（健康/心理/金钱/行动点）是短期消耗品，波动剧烈。这种分离让"能力"和"状态"有清晰的读写边界——属性通常只增不减（里程碑式成长），资源则每回合都在变动。

2. **终结态作为状态机而非直接死亡**：传统 RPG 的 HP=0 即死亡被替换为**终结态状态机**。当资源归零时，角色进入一个带倒计时的 TerminalState 对象（濒死/崩溃/匮乏），倒计时由对应属性计算（体魄决定濒死窗口、智力决定崩溃窗口）。这个设计的实现意义是：需要一套独立的状态流转逻辑（进入→每回合效果→脱离/超时→结局），而非简单的数值判定。

3. **角色与事件系统的双向校验**：事件执行前，角色系统提供 `preEventCheck()` 预测执行后的状态（包括是否会进入终结态），这让 UI 可以在玩家确认前显示预警；事件执行后，角色系统的 `applyEventEffects()` 统一应用数值变化并触发终结态检查。这种**预测-执行-检查**的三段式流程确保了数值一致性。

#### 事件系统：四类事件与槽位匹配引擎

事件系统是游戏的**驱动力**，技术架构需要支撑 169+ 个事件的灵活扩展和高效调度。

**四种事件类型的技术差异**：

| 类型 | 触发方式 | 数据结构核心 | 设计原因 |
|------|---------|------------|---------|
| **随机事件** | 回合开始时加权随机 | weight + conditions + cooldown | 给予玩家在游玩过程中的非常基础的随机感 |
| **固定事件** | 玩家主动点击 | actionPointCost + slots + repeatability | 玩家的整个游玩经历和过程需要一系列固定事件进行推动 |
| **事件链** | 前置节点完成 | prerequisites + branches | 需要链式状态追踪和分支条件判定，支持跨场景持久化，用于实现复杂事件流程 |
| **政策压力事件** | 回合开始（场景限制） | scene-specific + one-time | 仅场景3触发，需要独立的事件池和场景绑定，并且需要这样独立的事件作为场景3中的压力来源 |

**场景2强制负面事件**：技术上这是一个特殊的随机事件子集，每回合开始时绕过普通随机事件池，直接从一个权重固定的强制事件池中选取（劫匪20%、毒蛇20%、小径20%、沼泽15%、毒虫15%、食物10%）。失败即调用 `triggerGameOver()`，这种**强制触发+即死判定**的机制在代码层面需要与普通随机事件的"可选执行"做明确区分。未来在场景3下，也会出现类似的，有着相同效果或者给玩家类似体验的事件，只不过文本描述和效果不同。

#### 道具系统：标签驱动的装备匹配
道具系统的核心是给予玩家在回合内，不消耗行动点的选择，并且这样的选择权是跨回合的，以便增强玩家在整个场景周期内的策略深度，扩展玩法可能性。
**两种存储结构的差异**：

- **消耗品**：用完就会被消耗，给予一次性的单回合的提升，用于临时增强角色以应对各种各样的突发事件，或者在关键节点获得超额收益以滚起雪球。
- **常驻道具**：用于推进游戏故事，或者是解锁更多事件，解锁更高收益的道具，属于长期内容，玩家依靠常驻道具，在当前回合消耗资源，但是换取长期回合的更高的收益，也是用于滚雪球的办法，但是收益曲线和消耗品不同。

**场景切换清空常驻道具栏**：非法移民跨境时通常会失去一切装备和关系网，真实情况下，其实每跨越一个大场景，也是很难保留身上的绝大部分物品的；每个场景都是独立的生存挑战（Rogue-like 体验）；上一场景是"经验积累"，新场景是"全新开始"。唯一的例外是全局书籍池——跨场景保留作为整局游戏的战略资源。

**书籍**：作为玩家的属性的永久提升，玩家可以在当前场景中使用而提升属性，从而达到书籍能力跨场景保留的实际效果。书籍池从预制项目中随机读取，不会全部读取，保证每局游戏的读书体验有差异，也同时防止在多个重复游玩回合中，玩家在不同周目的同一个阶段重复获取同一本书。

#### 场景系统：单向流与独立事件池

场景系统的核心约束是**单向流动**（act1 → act2 → act3，不可返回）和**完全独立**（每个场景拥有独立的事件池、道具栏和状态）。

**技术实现**：

- **场景运行时数据** `SceneRuntimeData` 包含：回合计数、场景特有状态（如场景2的雨林进度、场景3的签证状态）、已触发/已完成事件集合、进行中的事件链、场景切换进度。
- **独立事件池**：`SceneEventPool` 为每个场景维护独立的 randomPool、fixedPool、chainPool 和 terminalEvents。场景切换时旧池被冻结，新池从配置初始化。这种隔离确保了场景间不会互相污染事件状态。
- **跨场景数据 `CrossSceneData`**：只携带书籍池、进行中的跨场景事件链、关键剧情道具（`cross_scene` 标签）、累计统计、场景2债务。常驻道具栏被清空，但金钱保留——技术上金钱存储在角色资源中，不属于道具系统。

**场景切换检定**：每条路径（走线/旅游签/学生签/货车偷渡/沙漠缺口/攀爬边境墙）都有独立的 `TransitionCheck` 配置，包含：检定方式（直接购买/属性检定/完成事件链/特定事件触发）、基础消耗、要求条件、失败后果（包括 `gameOver: true` 和 `healthLoss: 999` 的即死标记）。

**债务跨场景处理**：场景1→2时债务清零（叙事上"国内的债主再也找不到你了"），场景2→3时债务保留（"蛇头的债是跨国界的"）。技术上通过 `handleDebtTransition()` 在切换时分别处理——场景1债务直接清空，场景2债务写入 `CrossSceneData.carriedDebt` 并在新场景恢复。

#### 结局系统：统一终局与个性化叙事

结局系统是所有游戏结束路径的**统一收口**。无论死亡、通关还是特殊结局，都通过 `EndingManager.checkEnding()` 判定并触发。

**结局类型与判定优先级**：

```
死亡结局（DEATH）> 通关结局（SUCCESS）> 特殊结局（SPECIAL）
```

- **死亡结局**在回合开始/结束时检查终结态倒计时，倒计时归零即触发。终结态类型直接映射到死亡结局（濒死→体力耗尽、崩溃→精神崩溃、匮乏→贫病交加）。
- **通关结局**在事件链完成时检查，如庇护申请链完成后判定是否满足 `intelligence >= 10` 等条件。
- **特殊结局**包含独特的"放弃"（场景1停留30回合）和"彻底黑户"（场景3生存50+回合且生存能力≥15，isGameOver: false 可继续游玩但无法通关）。

**硬性约束的技术实现**：

| 设计约束 | 设计结局的理由 |
|---------|---------|
| 入狱即终局 | 类似于主角这样的无合法身份的移民，入狱后便永无拿到身份的可能，也就无法通向Good Ending。 |
| 场景3二元结局 | 结局判定条件中限制 `scene: 'act3'`，且只有"留在美国"和"死亡"两种路径 |
| 单结局原则 | `hasEnded` 布尔标志确保一个角色实例只触发一次结局 |

**结局统计**：结局时生成 `EndingStatistics`，包含总回合数、资源峰值、属性成长、经济收支、进入终结态次数、死里逃生次数等。这些数据同时写入存档支持多周目回顾。

#### 存档系统：适配器模式与版本迁移

存档系统采用**适配器模式**（Adapter Pattern），统一接口 `IStorageAdapter` 支持 localStorage、IndexedDB 和云端存储切换。

**技术决策**：

1. **localStorage 为主，IndexedDB 为预留**：浏览器环境优先使用 localStorage（5MB 限制），大存档自动分片存储（每片1MB）。IndexedDB 适配器已实现但默认不启用，为未来云存档留扩展点。

2. **存档压缩**：序列化后的 JSON 超过 100KB 时自动压缩（预留 LZ-String 接口），减少存储占用。

3. **版本迁移**：存档数据包含 `dataVersion` 和 `migrationHistory`，游戏更新后通过迁移脚本将旧存档升级到新格式，确保兼容性。

4. **自动保存 + 手动槽位**：5个手动存档槽 + 3个自动存档槽（FIFO 轮换），自动保存触发时机为每回合结束或关键事件后。

#### 债务系统：场景绑定的经济压力引擎

债务系统是独立的**经济压力模块**，与场景系统紧密耦合。通过债务系统，给予玩家在失误的时候，一次挽回游戏进程的机会，但是伴随而来的则是更加巨大的压力。债务会像滚雪球一样压迫玩家的选择空间，直到最后没有通向 Good Ending 的路径。

**核心设计**：

- **分期还款**：每回合自动从现金中扣除 `repaymentPerTurn`，直至 `remainingTurns` 归零。如果现金不足，进入违约状态。
- **渐进惩罚**：第一次违约触发警告事件（心理-10），第二次违约没收道具，第三次违约触发暴力事件（健康-20）。这种**阶梯式惩罚**在技术上通过 `defaultHistory.count` 计数器实现。
- **场景特定利率**：债务利率是固定独立数值，不与场景3的通胀系统挂钩。场景1利率 10-30%/月（高利贷），场景2紧急借贷 50%/月，场景3信用卡 5-15%/月。

---

#### 统计系统：惰性收集与跨游戏聚合

统计系统采用**惰性收集策略**——不实时计算复杂指标，只在关键节点（事件完成、回合结束、结局触发）记录原始数据，需要展示时再做聚合。

**统计分类**：基础（回合/时间/场景切换）、经济（收入/支出/通胀/生活成本）、事件（完成数/链进度/死亡原因）、结局（达成次数/难度分布）、角色（属性成长/资源极值/终结态次数）。

**多周目支持**：单局统计数据写入存档，跨游戏的累计统计单独存储（如 `total_play_time`、`ending_unlock_count`），为成就系统和解锁系统提供数据源。

---

#### 游戏配置中心：数据驱动的单一数据源

配置中心是**策划与开发的桥梁**。所有平衡性数值（属性成长速率、事件权重、生活成本基准、通胀倍率、终结态公式）集中在 `GameConfig` 接口中，按系统分层（character/event/item/scene/debt/economy）。

**技术特性**：

- **热重载**：开发模式下修改配置无需重启游戏（通过动态导入实现）。
- **环境隔离**：开发/测试/生产环境使用不同配置覆盖，支持 easyMode/hardMode/extremeMode 快速切换。
- **类型安全**：TypeScript 接口约束确保配置格式正确，编译时即捕获错误。

### DesignDocs — 游戏设计文档

#### 角色系统：为什么这样设定属性

游戏采用**单主角制**，玩家与角色绑定体验完整的"一生"，死亡即游戏结束、不可复活。角色拥有六项核心属性，每一项都对应着非法移民生存中无法回避的真实能力缺口：

| 属性 | 现实中的映射 | 设计意图 |
|------|------------|---------|
| **体魄** | 体力、耐力、负重能力 | 对应高强度黑工（装修、搬运）和跨境徒步的生存基础。高体魄延长濒死窗口，奖励玩家对身体素质的投入 |
| **智力** | 学习能力、信息处理、骗局识别 | 对应准备假材料、识别中介骗局、理解移民法规。高智力反而缩短崩溃窗口——**聪明人的诅咒**，清醒地看着自己堕落 |
| **英语技能** | 语言沟通能力 | 决定玩家能否从"华人区黑工"升级到"面向本地人的工作"，是场景3阶层跃迁的关键门槛 |
| **社交** | 人脉、信息渠道、口碑 | 对应社群中的口耳相传：谁有工作、谁有房源、谁可靠。社交高时更容易触发正面随机事件，能够触发更多的事件，但也更容易引来麻烦 |
| **风险意识** | 对危险的预判和止损能力 | 对应非法移民每天都在做的风险计算：这条路安不安全、这个中介靠不靠谱、要不要跑 |
| **生存能力** | 极端环境下的应变与钻制度空子的能力 | 场景2的核心属性，也是场景3中"用技巧换资源"的底层逻辑 |

**设计核心**：没有"完美角色"，只有"代价选择"。高体魄 Build 不怕濒死但容易因忽视智力而崩溃；高智力 Build 心理稳定但身体可能扛不住。这种制衡迫使玩家在每次决策中权衡取舍。

#### 事件库：三幕结构中的 169+ 个事件

事件库按三幕场景组织，每个场景的事件设计服务于该幕的核心体验目标：

**场景1（国内准备，约40个事件）——"你会选择留下吗？"**

场景1是新手教学+决策准备期，设计上**很难死亡**，但设置了独特的"放弃"机制：当玩家在国内停留超过30回合，会触发特殊事件"放弃幻想"——承认留在国内也是一种选择，视为通关一次。这不是惩罚，而是设计上的**元反思**：让玩家意识到"去美国"是一个主动的选择而非必然。

场景1的事件围绕"打工攒钱→属性提升→准备离境"的循环展开：
- **主动事件**：快递分拣、送外卖、工地短包、健身房训练、读书学习、水群社交、认识蛇头、兑换美元
- **随机事件**：房东涨降租、水群遇骗子、警察上门调查、朋友送别、家庭冲突
- **设计意图**：让玩家在安全环境中建立对资源管理的基本认知，同时为场景2的不可逆决策积累筹码

**场景2（跨境穿越，约40个事件）——"一旦出发，就没有回头路"**

场景2是**死亡概率最高的场景**，设计上强制**单向流动**——不能返回场景1，模拟真实走线的不可逆性。事件分为两个阶段：

- **阶段1（雨林/荒野）**：每回合强制触发一个环境致命事件——劫匪、毒蛇、危险小径、沼泽、毒虫、可疑食物。玩家必须通过属性检定（体魄、风险意识、生存能力）来避免死亡。跟随向导可大幅降低风险，但需要持续支付费用；独自穿越高风险高回报
- **阶段2（边境小镇）**：玩家需要在小镇中补充物资、打探消息、联系货车司机，然后选择穿越方式（货车偷渡、沙漠缺口、攀爬边境墙）
- **设计意图**：制造"生存恐惧"——资源在消耗、危险在逼近、每一步选择都可能是最后一步。场景2的死亡不是"游戏结束"的惩罚，而是对玩家决策重量的诚实反馈。同时因为其流程较短，也是作为场景3的前奏与教学，为玩家提供一个简短的环境快速上手生存系统，明白自己需要管理角色的生理健康和心理健康值。

**场景3（美国生存，约74个事件）——"结构性压迫下的慢性消耗"**

场景3的核心体验是**现金流断裂焦虑**。与场景2的"突发死亡"不同，场景3是"慢性死亡"——玩家被困在一个结构性恶性循环中：

```
需要钱生存 → 打工赚钱 → 身体/心理消耗 → 健康下降 → 
需要更多钱治病/恢复 → 更多打工 → 身体更差 → 死亡
```

场景3的事件设计围绕四个压力维度展开：
1. **现金流压力**：每月固定扣除生活成本（食品$400+住宿+出行），通胀事件导致成本上涨，收入却因账号被封、被抢劫、突击检查而充满不确定性
2. **身份压力**：签证逾期倒计时、ICE突击检查（随机触发，与身份状态无关——正如现实中ICE也会误抓公民）、假证件被识破风险
3. **身体消耗**：强化剂过度使用致死、长期过劳、小病拖成大病（无医保）
4. **心理压力**：孤独、恐惧、身份焦虑、无法与家人联系

场景3还包含三条绿卡通关路径：
- **金卡路径**：现金≥$1,000,000直接购买（讽刺性地映射现实政策）
- **政治庇护路径**：事件链驱动，总成本$2000+、耗时15回合、成功率取决于材料质量和运气
- **U签证路径**：需要先成为犯罪受害者获得证明，再申请签证

> 📄 完整事件索引见 [DesignDocs/Events/EventPool.md](./DesignDocs/Events/EventPool.md)

#### 道具系统：连接事件与属性的枢纽

道具不是简单的"装备加数值"，而是整个游戏**事件-槽位匹配系统**的核心枢纽。道具通过**属性标签**与事件的**槽位**动态匹配，决定了玩家能执行哪些事件、以及事件执行的效率。

**道具的两种类型**：

| 类型 | 说明 | 代表道具 |
|------|------|---------|
| **消耗型** | 一次性使用，回复资源或增加属性 | 止痛药、大麻、强化剂（I/II/III级）、黑市抗生素、能量饮料 |
| **常驻型** | 持续生效，通过槽位匹配影响事件 | 交通工具、住宿钥匙、假证件、手机、向导服务 |

**道具的核心作用**：

1. **解锁事件选项**：没有 `transport` 标签道具就无法送外卖；没有 `identity` 标签道具就无法做Uber司机。道具决定了玩家的"可行选择空间"
2. **改变事件效率**：同样是送外卖，二手代步车收入×1.0、特斯拉收入×1.3；同样是住宿，收容所免费但ICE突袭风险+10%、公寓$1200/月但心理+5/晚
3. **提供策略纵深**：玩家需要在有限的常驻道具栏位（6~8格）中做出取舍——是带武器防身还是带水袋保命？是住便宜的家庭旅馆还是租正规公寓？
4. **承载叙事信息**：道具描述本身就是世界观的一部分——"一套包含假SSN、假护照、假驾照的证件套装，照片是你，信息是别人"、"一张写着电话号码的纸条，据说他的货柜里有夹层"

**场景2专属生存道具**（水袋、急救包、砍刀、指南针、睡袋）和**场景3购物App系统**（CarGo购车、PillNow买药、ID Helper办证）进一步将道具融入场景主题，让玩家在每个阶段都面临不同的资源管理挑战。

#### 玩法机制：回合制与终结态

**回合流程**：游戏采用六阶段回合制——状态恢复 → 环境压力结算 → 随机事件判定 → 玩家行动 → 资源结算 → 终结态检查。每个场景的时间尺度不同：场景1每回合代表2-3天（允许长期规划），场景2每回合代表0.5-1天（紧迫感），场景3每回合代表1天（日常节奏）。

**终结态机制**：这是游戏最核心的压力系统。当关键资源降至零时，角色不会立即死亡，而是进入一个**带倒计时的警告状态**：

| 终结态 | 触发条件 | 核心设计 |
|--------|---------|---------|
| **濒死** | 身体健康度 ≤ 0 | 倒计时窗口由体魄决定（0~3回合）。窗口内玩家有机会通过急救恢复，但每回合行动点被额外扣除，且濒死期间获得的debuff不会自动消失 |
| **崩溃** | 心理健康度 ≤ 0 | 倒计时窗口由智力决定（1~3回合）——**智力越高窗口越短**。这是"聪明人的诅咒"的设计具象化。倒计时归零后直接游戏结束 |
| **匮乏** | 资金 ≤ 0 | 软性惩罚，不直接致死，但每回合侵蚀健康和心理，触发借高利贷、街头乞讨、翻垃圾桶等专属事件 |

终结态的设计意图是**"警告而非惩罚"**——给玩家喘息窗口来调整策略，同时通过"脱离后保留debuff"的机制传递一个信息：你只是暂时活过来了，下次不会这么幸运。

**经济系统**：场景3引入月度固定支出机制（食品$400+住宿+出行），制造"现金流断裂"的核心焦虑。收入却充满不确定性——Uber账号可能被封、假证件可能被识破、ICE突袭可能导致停工。玩家必须在"省钱"和"冒险"之间做选择。

#### 世界机制：场景特有的环境压迫

每个场景都有独特的**环境Debuff**，作为背景压力持续作用：

| 场景 | Debuff | 效果 |
|------|--------|------|
| 场景1 | 【就业冰河期】 | 经济下行，打工收入下降 |
| 场景2 | 【恶劣的环境】 | 每回合大幅扣除健康，物资损耗加快 |
| 场景3 | 【大通胀时代】+【移民搜捕升级】 | 现金消耗递增、基础心理消耗；ICE突击检查概率增加 |

**执法压力系统**：全局共享的压力值，只增不减，每回合基础+2。压力等级从LOW到CRITICAL，影响随机事件的权重分布——高压力时负面事件概率上升、正面事件概率下降。这是“大环境恶化，政府和民间开始系统性压迫非法移民”的游戏化表达。

**场景切换的不可逆性**：场景1→2时国内债务清零（"那些债主再也找不到你了"）；场景2→3时蛇头的债跨国追讨（"蛇头的债，是跨国界的"）。这种设计强化了每个决策的重量。同时用于体现场景1的教学目的，一些错误的抉择会得到惩罚，但是程度非常的轻以至于不太会影响游戏进程。

---

#### 世界观：冷静克制的黑色幽默

世界观设定不是背景装饰，而是所有事件文案、道具描述、角色互动的**底层语境**。

**叙事基调**：第二人称（"你"）、冷静克制的旁观者视角、带黑色幽默。严禁直接显示数值——"健康度-30"大体上会写成"你感到一阵眩晕"。这种"数值包装化"强制玩家通过**体感描述**而非数字来理解角色状态。

**核心主题**：游戏讨论的是"去美国"这一选择的代价与荒诞，以及真正能够在美国落脚站住的人的选择可能。场景1的安稳平淡"没有梦想，但是追寻美国作为梦想"，场景2/3的危险压迫却被称为"自由"。文案中藏着现实移民论坛的黑话、电子宠物的金句、推特上的荒诞政策——"甜甜圈真好吃""自由的气息""来了就是美国人"——不直接解释，让玩家自己体会。

**阵营与势力**：
- **蛇头网络**：从国内"出国顾问"到中美洲向导再到跨国走私客，每个层级都有卷款跑路、抛弃、出卖的风险
- **ICE/CBP**：场景3中随机触发的突击检查，与玩家身份状态无关——正如现实中移民执法的不可预测性
- **华人地下经济**：中餐馆、装修队、美甲店、Uber 外卖/网约车——这些是场景3打工事件的现实原型

**世界设定文档**（`World/`）提供了地点考据（蒂华纳-圣地亚哥、华雷斯-埃尔帕索等真实偷渡点）、移民政策历史（从1986年IRCA到特朗普"零容忍"政策）、跨境路线危险数据（达连雨林2022年约25万人穿越）等支撑材料，确保叙事在荒诞中保持真实感。游戏中的场景2和场景3事件大量参考真实新闻报道，但是做出符合游戏玩法的加工。

#### 叙事系统：事件驱动与标记追踪

叙事不由传统对话树主导，而是通过**事件驱动+叙事标记**自然推进。玩家的每个关键选择都会设置 Story Flag，影响后续事件的触发和文本展示：

- 是否在场景1交了蛇头定金，以及场景1出国的具体选项 → 影响场景2的过渡选项，以及是否会经过场景2
- 是否全额支付律师费 → 影响庇护申请事件链的文本和成功率
- 是否背叛过同伴/祖国 → 影响结局文案的追加段落

这种设计让每个玩家的游戏历程都是**个性化的叙事弧线**，结局文案会根据玩家的选择动态生成——"你曾经答应家人要出人头地"，"你时常想起那个被你出卖的朋友"。
