# NarrativeDocs/Dialogues/ 对话文档总览

> 本文档目录包含《去美国》游戏的所有对话文本和叙事文案，供文案撰写人员参考使用。

---

## 1. 目录用途

本目录存放游戏的**叙事文本内容**，与 `DesignDocs/Events/` 中的机制设计文档相对应：

| 文档类型 | 存放位置 | 内容说明 |
|---------|---------|---------|
| **机制设计** | `DesignDocs/Events/` | 事件结构、数值配置、触发条件、槽位系统 |
| **叙事文本** | `NarrativeDocs/Dialogues/` | 事件描述、选项文本、结果文案、对话内容 |

> **工作流程**：先在 `DesignDocs/` 中设计事件机制，然后在本目录中撰写对应的叙事文本。

## 3. 写作规范

### 3.1 事件描述格式

每个事件文案必须包含以下四个部分：

```markdown
### 事件名称 (ID: `event_id`)

**你看到了什么**（场景描写，50-100字）

用第二人称描述玩家看到的场景，营造氛围。避免直接说明数值变化，专注于感官体验和情绪渲染。

**可选行动**：
1. **选项A**（条件：属性≥X / 需道具：xxx）→ 结果描述 + 数值变化
   - 选项文本：10-20字，简洁有力
   - 结果文本：30-50字，描述执行后的体验和后果
   
2. **选项B** → 结果描述 + 数值变化

3. **选项C** → 结果描述 + 数值变化
```

### 3.2 文案长度规范

| 元素 | 长度限制 | 说明 |
|-----|---------|------|
| 事件标题 | 2-6字 | 简洁明了，如"超市夜班"、"暴雨袭击" |
| 场景描述 | 50-100字 | 建立氛围，提供决策背景 |
| 选项文本 | 10-20字 | 玩家可点击的按钮文字 |
| 结果描述 | 30-50字 | 执行后的体验和后果描述 |
| 特殊提示 | 15-30字 | 解锁/条件不满足时的提示 |

### 3.3 数值包装原则

**❌ 禁止直接显示数值**：
- ~~身体健康度 +10~~
- ~~心理健康度 -5~~
- ~~现金 +500~~

**✅ 使用体验化描述**：

| 数值变化 | 文案表达 |
|---------|---------|
| 身体健康度 +10 | "感觉身体恢复了些许"、"腰背舒展了一些" |
| 身体健康度 -10 | "一阵眩晕袭来"、"身体抗议着这份疲惫" |
| 身体健康度 -20 | "剧烈的疼痛让你弯下了腰"、"视线开始模糊" |
| 心理健康度 +10 | "心情好了一些"、"心头的一块石头落地了" |
| 心理健康度 -10 | "感到一阵焦虑"、"不安在心底蔓延" |
| 心理健康度 -20 | "绝望感如潮水般涌来"、"你开始怀疑一切" |
| 现金 +500 CNY | "钱包里多了几张钞票"、"收入囊中" |
| 现金 -500 CNY | "数出几张钞票递过去"、"钱包瘪了一些" |
| 现金 +100 USD | "几张绿票子进了口袋"、"美元到手" |
| 现金 -100 USD | "数出几张绿票子"、"钱包又瘦了" |
| 行动点 -1 | "时间过去了"、"一个时段结束了" |
| 行动点 -2 | "这花了你不少时间"、"大半天的工夫" |
| 终结态倒计时 | "你能感觉到情况在恶化"、"时间不多了" |

## 5. 文案风格指南

### 5.1 叙事语气

- **基调**：冷静克制的旁观者视角，带黑色幽默
- **视角**：第二人称（"你"）
- **人称**：避免直接说教，让事件本身说话
- **情感距离**：不过度煽情，也不过度疏离

### 5.2 场景语气差异

#### 场景1：国内准备
- **语气**：现实主义、疏离感、黑色幽默
- **情绪**：焦虑与期待并存、对未来的不确定
- **关键词**：打工、攒钱、告别、准备
- **示例**："夜班结束，你从超市里走出来，天已经亮了。八十块钱到手，腰背像是被拆散重组过一遍。"

#### 场景2：跨境穿越
- **语气**：紧张、生存压力、原始恐惧
- **情绪**：恐惧、绝望、偶尔的希望
- **关键词**：荒野、危险、同伴、抉择
- **示例**："暴雨倾盆而下，你们挤在一棵树下，雨水顺着领口灌进去。向导骂着脏话，说这是雨季的第一场。"

#### 场景3：美国生存
- **语气**：荒诞、讽刺、慢性焦虑
- **情绪**：被追捕的紧张、生存的疲惫、黑色幽默
- **关键词**：账单、盘查、黑工、身份
- **示例**："唐纳德总统又发推特了。你看着手机上的翻译，虽然没全看懂，但那个大写的DEPORT你认识。"

### 5.3 金句与梗原则

- **来源**：推特/现实中的电子宠物事迹、论坛黑话
- **呈现**：藏在事件描述或选项文本中，不直接解释
- **传播性**：单句可截图，有共鸣感
- **示例**：
  - "正在学英语，准备考托福。"（讽刺盲目乐观）
  - "自由的气息。"（场景3初到的苦涩幽默）
  - "甜甜圈真好吃。"（引用网络meme）

### 5.4 避免的表达方式

❌ **禁止**：
- 直接说教或道德评判
- 过度详细的数值说明
- 打破第四面墙的meta评论
- 过于书面化的表达
- 重复用词（每个场景建立关键词库，保持用词多样性）

✅ **推荐**：
- 用感官细节暗示状态（"口袋里只剩几个硬币的声响"）
- 通过行为暗示心理（"你数了三遍剩下的钱"）
- 环境描写映射内心（"阴雨天持续了整整一周"）

## 6. 文档维护

### 6.1 更新流程

1. **机制变更时**：当 `DesignDocs/` 中的事件配置变更时，同步更新对应文案
2. **新增事件时**：先在 `EventPool.md` 登记，然后在对应场景的 Dialogues 文件中撰写文案
3. **文案优化时**：保持事件 ID 不变，仅修改描述文本

### 6.2 版本标记

在文案底部添加版本信息（可选）：

```markdown
---
*文档版本: v1.0*
*最后更新: 2026-02-27*
*对应机制版本: EventSystemArchitecture v1.2*
```
