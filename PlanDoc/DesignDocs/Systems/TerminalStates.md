# 终结态机制（Terminal States）

## 概述

终结态是《去美国》的核心压力机制。当角色的关键资源降至零时，将进入特殊状态，面临持续性的负面效果和紧迫的倒计时压力。

三种终结态：
- **【濒死】**：身体健康度 ≤ 0
- **【崩溃】**：心理健康度 ≤ 0  
- **【匮乏】**：资金 ≤ 0

---

## 设计意图

### 核心体验目标
1. **警告而非惩罚**：给玩家喘息窗口，提醒其调整策略
2. **属性投资的回报**：高体魄/智力可延长窗口期，体现属性价值
3. **滚雪球压力**：不处理会迅速恶化，制造紧迫感
4. **记忆点制造**："上次差点死了"的体验增强游戏记忆

### 与传统 HP 系统的区别
| 传统 RPG | 本游戏终结态 |
|---------|-------------|
| HP=0 直接死亡 | 进入终结态，有倒计时窗口 |
| 治疗即恢复 | 脱离后保留 debuff，"下次不会这么幸运" |
| 死亡=失败 | 终结态=警告，给予最后机会 |

---

## 【濒死】终结态

### 触发条件
身体健康度 ≤ 0

### 即时效果（进入时触发）
- 行动点归零（或强制设为 0）
- 触发"濒死冲击"：额外扣除少量金钱/物资（代表急救成本）
- 记录状态：`终结态 = 濒死`

### 倒计时窗口计算

```
倒计时上限 = clamp( floor((体魄 + 6) / 7), 0, 3 )
```

| 体魄值 | 倒计时上限 | 说明 |
|-------|-----------|------|
| 0-1 | 0 回合 | 立即死亡 |
| 2-8 | 1 回合 | 极短窗口 |
| 9-15 | 2 回合 | 标准窗口 |
| 16-20 | 3 回合 | 最大窗口 |

### 每回合流程

#### 回合开始
1. **不触发随机事件**（终结流程接管）
2. **行动点恢复受限**：恢复上限被严重压低
3. **额外扣除**：每回合开始时额外扣除 1 点行动点（等差数列累加）
   - 第1回合：-1
   - 第2回合：-2
   - 第3回合：-3

#### 回合结束
若身体健康度仍 ≤ 0：
- 进入下一回合时添加 debuff（失血/发烧/抽筋）
- 额外扣除心理健康度（恐惧与痛觉）

### 脱离条件
身体健康度恢复至 > 0

### 脱离后果
- 终结态解除
- 濒死期间获得的 **debuff 不会自动消失**
- 提醒文案："你只是暂时活过来了，下次不会这么幸运"

### 死亡判定
当倒计时超过上限值时，游戏结束，触发死亡结局。

---

## 【崩溃】终结态

### 触发条件
心理健康度 ≤ 0

### 设计逻辑
崩溃不是"惩罚"，而是"判决的宣读"。智力越高的人，崩溃时自我毁灭越快（清醒地看着自己堕落）。

### 即时效果（进入时触发）
触发"崩溃冲击"（随机选择一项）：
- 随机丢失一个道具
- 百分比降低随机一项可消耗资源
- 降低一个随机属性值 5 点

记录状态：`终结态 = 崩溃`

### 倒计时窗口计算

```
崩溃窗口回合 = clamp( 3 - floor(智力 / 10), 1, 3 )
```

| 智力值 | 窗口回合 | 设计意图 |
|-------|---------|---------|
| 0-9 | 3 回合 | 迟钝反而缓冲了崩溃 |
| 10-19 | 2 回合 | 标准窗口 |
| 20 | 1 回合 | 聪明人的诅咒，清醒着毁灭 |

### 每回合流程

#### 回合开始
1. **不触发随机事件**
2. **行动点恢复受限**：恢复上限被压低
3. **额外扣除**：每回合开始时额外扣除 1 点行动点（过度思考/惊恐反刍/注意力涣散）
4. **强制触发"崩溃冲击"事件**

#### 回合结束
若心理健康度仍 ≤ 0：
- 倒计时 -1
- 下一回合开始前强制追加"高负面收益事件"（通常带自毁倾向）

### 脱离条件
心理健康度恢复至 > 0

### 脱离后果
- 终结态解除
- 崩溃期间丢失的属性和资源**不会恢复**
- 提醒文案："你只是暂时活过来了"

### 游戏结束判定
倒计时归零后若心理仍 ≤ 0，**直接游戏结束**。

结局呈现：自毁性死亡 / 被捕 / 失踪（以文案呈现，不显示数值）

## 【匮乏】终结态

### 触发条件
- 场景1：人民币 ≤ 0
- 场景2/3：美元 ≤ 0

### 与其他终结态的区别
匮乏是**软性惩罚**，不会直接导致游戏结束，但会持续侵蚀其他资源。

### 持续效果
- 每回合扣除身体健康度（-5/回合）
- 每回合扣除心理健康度（-5/回合）
- 部分打工事件灰色不可点击
- 触发匮乏专属随机事件（借高利贷、街头乞讨等）

### 匮乏专属事件池

| 事件ID | 名称 | 效果 |
|-------|------|------|
| `term_loan_shark` | 借高利贷 | 现金+500，但陷入债务陷阱 |
| `term_beg` | 街头乞讨 | 现金+20~80，社交-2 |
| `term_desperation` | 出卖尊严 | 现金+100，心理-30 |
| `term_dumpster` | 翻垃圾桶 | 健康-10，心理-20 |

### 脱离条件
资金恢复至 > 0

---

## 终结态叠加规则

理论上三种终结态**可以同时存在**：
- 资金匮乏 → 健康/心理持续下降
- 健康归零 → 进入濒死
- 心理归零 → 进入崩溃

此时需同时满足三个脱离条件才能恢复正常。

## 文案表达规范

### ❌ 禁止直接显示数值
```
错误："心理健康度 -5，倒计时 2 回合"
```

### ✅ 用叙事包装
```
濒死：
- "你感到意识模糊，每一次呼吸都伴随着剧痛"
- "血从绷带渗出，你知道撑不了多久了"
- "视线开始发黑，耳边是心跳的轰鸣"

崩溃：
- "你的思绪像断线的风筝，抓不住任何一点理性"
- "一种冰冷的绝望从心底升起"
- "你开始怀疑这一切是否还有意义"

匮乏：
- "口袋空空如也，连明天的早餐都没有着落"
- "你开始计算哪些东西可以卖掉换钱"
```

### UI 提示
- 使用红色边框/闪烁效果提示危险状态
- 倒计时用"危"、"急"、"危殆"等汉字表示严重程度
- 状态栏显示 debuff 图标，悬停显示描述

## 平衡性参考

### 濒死窗口期望
- 普通玩家（体魄 5-10）：1-2 回合窗口
- 偏科玩家（体魄 15+）：2-3 回合窗口
- 脆弱角色（体魄 <5）：0-1 回合窗口，高风险

### 崩溃窗口期望
- 普通玩家（智力 5-10）：2-3 回合窗口
- 高智角色（智力 15+）：1-2 回合窗口，高风险
- 低智角色（智力 <5）：3 回合窗口，相对安全

### 设计原则
濒死和崩溃应对**不同属性流派**形成制衡：
- 高体魄 Build：不怕濒死，但可能因忽视智力而容易崩溃
- 高智力 Build：心理稳定，但身体可能扛不住
- 均衡 Build：两者都有适中窗口，最安全但上限较低
