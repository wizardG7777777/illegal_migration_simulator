# 回合流程（Turn Flow）

## 概述

《去美国》采用**回合制**推进游戏进程。每个回合代表一段时间（根据场景不同，时间尺度不同），玩家有固定的行动点用于执行事件。

---

## 回合结构总览

```
回合开始
    │
    ├──▶ [阶段1] 状态恢复
    │       ├── 行动点恢复
    │       └── 持续效果结算（debuff/buff）
    │
    ├──▶ [阶段2] 环境压力结算
    │       └── 场景特有环境 Debuff 效果
    │
    ├──▶ [阶段3] 随机事件判定
    │       └── 触发 0-1 个随机事件
    │
    ├──▶ [阶段4] 玩家行动阶段
    │       └── 消耗行动点执行事件
    │
    ├──▶ [阶段5] 资源结算
    │       └── 固定支出扣除（场景3月度成本）
    │
    └──▶ [阶段6] 终结态检查
            ├── 资源归零检查
            └── 终结态回合流程（如适用）
                └── 游戏结束判定
```

---

## 各阶段详细说明

### 阶段1：状态恢复

#### 行动点恢复
```
当前行动点 = min(最大上限, 上一回合剩余 + 每回合恢复量)
```

| 场景 | 最大上限 | 每回合恢复 | 说明 |
|-----|---------|-----------|------|
| 场景1 | 4 | 4 | 标准恢复 |
| 场景2 | 3 | 3 | 恶劣环境，恢复受限 |
| 场景3 | 5 | 5 | 相对自由 |

#### 持续效果结算
- **debuff 持续时间 -1**
- **buff 持续时间 -1**
- 持续时间归零的效果被移除
- 剩余的 debuff 效果在本回合持续生效

---

### 阶段2：环境压力结算

#### 场景特有环境 Debuff

| 场景 | Debuff名称 | 效果 |
|-----|-----------|------|
| 场景1 | 【就业冰河期】 | 经济下行，打工收入下降，就业困难 |
| 场景2 | 【环境恶化】 | 每回合身体健康大幅扣除，物资损耗加快 |
| 场景3 | 【大通胀时代】 | 现金消耗递增、基础心理消耗 |

#### 场景3特有：执法压力结算
```
if (场景3 且 存在 pressure 类型 Debuff):
    根据强度等级应用效果:
    - 突击检查概率增加
    - 打工收入降低
    - 心理健康扣除
```

---

### 阶段3：随机事件判定

#### 触发条件检查
系统遍历所有可用的 `RANDOM` 类型事件：

```
for each randomEvent:
    if (满足场景限制 且 
        满足前置条件 且 
        不在冷却期 且 
        未达最大触发次数):
        
        加入候选池，权重 = event.trigger.weight
```

#### 加权随机选择
```
总权重 = sum(所有候选事件权重)
随机数 = random(0, 总权重)
选中事件 = 按权重累加后覆盖随机数的事件
```

#### 终结态覆盖
如果玩家处于终结态：
- **不触发普通随机事件**
- **改为触发终结态专属事件**

#### 事件执行
- 显示事件描述文案
- 展示选项（如果有）
- 应用事件效果
- 记录事件触发（用于冷却和次数限制）

---

### 阶段4：玩家行动阶段

#### 可用事件列表
玩家可执行的事件类型：

| 类型 | 说明 | 消耗 |
|-----|------|------|
| `FIXED` | 固定事件（打工/读书/购买） | 行动点 |
| `CHAIN` | 事件链的当前节点 | 行动点 |
| `CONSUMABLE` | 使用消耗型道具 | 无（通常）|

#### 事件执行流程
```
1. 玩家点击事件
2. 系统计算槽位匹配和最终数值
3. 显示确认界面（预计效果）
4. 玩家确认执行
5. 扣除行动点
6. 应用事件效果
7. 显示结果文案（不显示具体数值）
8. 事件计入已完成列表
```

#### 行动点耗尽
当行动点 ≤ 0 时：
- 玩家事件列表灰色不可点击
- 显示"你今天太累了，需要休息"
- 进入阶段5

---

### 阶段5：资源结算

#### 月度固定支出（仅场景3）

```
if (回合数 % 30 == 0 且 场景3):
    扣除月度生活成本:
        食品 = 基础$400 × 食品通胀率
        住宿 = 住宿道具成本 × 住宿通胀率
        出行 = 出行道具成本 × 出行通胀率
```

#### 终结态资源扣除
如果处于终结态：
- 【濒死】：额外扣除行动点上限，可能触发debuff
- 【崩溃】：扣除倒计时，触发崩溃冲击
- 【匮乏】：扣除健康/心理，触发匮乏事件

---

### 阶段6：终结态检查

#### 资源归零检查
```
if (身体健康度 <= 0 且 不在濒死态):
    进入【濒死】终结态
    
if (心理健康度 <= 0 且 不在崩溃态):
    进入【崩溃】终结态
    
if (资金 <= 0 且 不在匮乏态):
    进入【匮乏】终结态
```

#### 已在终结态时
```
if (在濒死态):
    倒计时检查
    if (倒计时 > 上限):
        游戏结束（死亡）
    else:
        继续濒死流程
        
if (在崩溃态):
    倒计时检查
    if (倒计时 <= 0):
        游戏结束（崩溃自杀/被捕）
    else:
        继续崩溃流程
        
if (在匮乏态):
    继续匮乏流程（不致死但持续恶化）
```

#### 脱离检查
```
if (濒死态 且 健康 > 0):
    脱离濒死，保留debuff
    
if (崩溃态 且 心理 > 0):
    脱离崩溃，不恢复已损失属性
    
if (匮乏态 且 资金 > 0):
    脱离匮乏
```

## 时间尺度

### 各场景回合时间设定

| 场景 | 每回合代表 | 设计意图 |
|-----|-----------|---------|
| 场景1 | 2-3 天 | 准备期，允许长期规划 |
| 场景2 | 0.5-1 天 | 跨境期，时间紧迫感 |
| 场景3 | 1 天 | 生存期，日常节奏 |

### 回合数参考

| 场景 | 建议回合数 | 说明 |
|-----|-----------|------|
| 场景1 | 15-30 回合 | 足够积累但不过长 |
| 场景2 | 5-15 回合 | 紧张刺激的穿越 |
| 场景3 | 60+ 回合 | 长期生存，通关需要 |

## 特殊流程

### 场景切换流程

```
触发场景切换事件
        │
        ▼
显示过渡剧情
        │
        ▼
清空常驻道具栏（场景2→3保留？根据设计）
        │
        ▼
给予起始物资包
        │
        ▼
初始化新场景状态
        │
        ▼
开始新场景第1回合
```

### 游戏结束流程

```
触发死亡/崩溃结局
        │
        ▼
显示结局文案（根据游戏历程个性化）
        │
        ▼
显示统计数据
        │
        ▼
返回主菜单 / 重新开始
```

### 通关流程

```
获得绿卡（或其他通关条件）
        │
        ▼
显示通关结局文案
        │
        ▼
显示完整数据统计
        │
        ▼
解锁多周目要素（如果有）
        │
        ▼
返回主菜单
```

## 设计原则

1. **流程清晰**：每个阶段有明确的视觉区分
2. **玩家控制**：行动阶段完全由玩家掌控
3. **不可跳过**：随机事件和结算阶段强制展示，不可跳过
4. **信息透明**：回合开始时告知本回合的环境影响
5. **节奏控制**：场景2回合短、压力大；场景3回合长、规划性强
