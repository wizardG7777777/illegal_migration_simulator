# 场景2专属消耗品设计

> 本文档定义场景2（跨境穿越）的专属消耗品。
> 基于 ItemSystemArchitecture.md 中消耗型道具的数据结构设计。

---

## 1. 概述

场景2的消耗品设计遵循以下原则：
- **生存导向**：所有道具都服务于跨境生存的基本需求
- **资源稀缺**：补给点稀少，道具管理是核心玩法
- **重量限制**：携带空间有限，需要取舍
- **场景专属**：场景切换后清空，不保留至场景3

## 2. 医疗类消耗品

| 属性 | 值 |
|-----|---|
| **ID** | `act2_med_water_tablets` |
| **名称** | 净水片 |
| **子分类** | medical |
| **最大堆叠** | 20 |
| **单价** | $2 / ￥15 |

**效果**：
- 将污水/河水转化为可饮用水（1单位）

**使用场景**：

## 7. 消耗品平衡数据汇总

|-----|--------------|-----|

- 基础价格 ×1.5（运输成本）
- 部分物资可能缺货

## 8. 设计说明

1. **补给稀缺性**：场景2中可购买补给的地点极少，鼓励玩家在场景1储备

2. **重量决策**：携带更多补给意味着负重，影响移动速度和遇险概率

3. **多用途设计**：道具尽量设计多重用途（如电池既可供电也可点火）

4. **风险替代**：净水片 vs 净水吸管——便宜但需处理 vs 贵但方便

5. **黑市元素**：部分强力道具（强化剂）只能黑市购买，体现灰色地带
