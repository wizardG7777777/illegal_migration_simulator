# 消耗型道具 (Consumables)

## 1. 系统规则

### 1.1 基础定义
消耗型道具是**一次性使用**的道具，使用后会被消耗或扣除使用次数。

### 1.2 使用机制
- **使用方式**：直接使用，**不消耗行动点**
- **使用效果**：直接回复可消耗资源 或 直接增加角色能力值
- **使用次数**：每个消耗型道具都有使用次数限制，**一般为 1 次**
- **消耗判定**：使用次数归零后，道具自动删除，视为已消耗

### 1.3 数据结构

```yaml
消耗型道具通用结构:
  ID: string           # 唯一标识符
  名称: string         # 显示名称
  说明: string         # 道具描述
  品类: enum           # 分类标签
  使用方式: string     # "直接使用"或"通过事件使用"
  使用效果: object     # 属性加成/资源回复
  
  # 新增：事件槽位匹配系统
  属性标签: [string]   # 用于事件槽位匹配的标签列表
  优先级: int          # 0-9，数字越小优先级越高，事件默认选优先级最高的
  
  当前次数: int        # 剩余可使用次数
  最大堆叠: int        # 最大堆叠上限
```

### 1.4 属性标签说明

属性标签用于**事件物品槽位匹配**。例如：
- `transport` - 交通工具
- `weapon` - 武器
- `medical` - 医疗物品
- `book` - 书籍类
- `identity` - 身份凭证
- `lodging` - 住宿场所
- `tool` - 工具类

## 3. 书籍池（Books Pool）

**设计理念**：整局游戏只有一个**全局书籍池**，通过「购买书籍」事件从中随机抽取。每本书都是**唯一的**，一旦被购买就从池中移除。

书籍使用**`book`**属性标签，用于`read_book`事件的物品槽位匹配。
