# 常驻型道具 (Permanents)

## 1. 系统规则

### 1.1 基础定义
常驻型道具是**不可消耗、不可直接使用**的道具，通过**被动效果**为角色提供长期增益或解锁特定内容，或通过**事件物品槽位**提供属性加成。

### 1.2 使用机制
- **使用方式**：不可主动使用，装备/持有即生效
- **生效逻辑**：
  - **被动增益类**：持续生效，直到道具被移除
  - **事件槽位类**：被事件调用时提供属性加成
  - **解锁类**：解锁特定事件选项，道具存在期间可用
- **耐久度**：常驻型道具**没有使用次数限制**
- **失去方式**：只能通过随机事件或特定剧情被删除/没收/丢失

### 1.3 道具栏规则
- **占用规则**：每个常驻道具**独立占用一个格子**，不可堆叠
- **数量限制**：常驻道具栏位有限（建议 6~8 个）
- **重复处理**：相同 ID 的常驻道具通常**不可重复获得**

### 1.4 数据结构（新版）

```yaml
常驻型道具通用结构:
  ID: string           # 唯一标识符
  名称: string         # 显示名称
  说明: string         # 道具描述
  品类: enum           # 增益/解锁/混合
  
  # 新增：事件槽位匹配系统
  属性标签: [string]   # 用于事件槽位匹配的标签列表
  优先级: int          # 0-9，数字越小优先级越高
  
  # 事件槽位效果（当道具被事件调用时生效）
  槽位效果:
    属性加成: object   # 如：{ 行动点消耗: -1, 收入: 1.3 }
    解锁选项: [string] # 解锁的事件选项ID列表
  
  可被删除: bool       # 是否可能被随机事件移除
```

### 1.5 属性标签体系

| 标签 | 说明 | 典型道具 |
|-----|------|---------|
| `transport` | 交通工具 | 二手车、特斯拉、电动车 |
| `lodging` | 住宿场所 | 公寓钥匙、家庭旅馆钥匙 |
| `identity` | 身份凭证 | 假SSN、工牌 |
| `weapon` | 武器 | 砍刀 |
| `tool` | 工具 | 指南针、手机、地图 |
| `guide` | 向导服务 | 墨西哥向导、走私客 |
| `membership` | 会员资格 | 健身月卡 |
| `document` | 文件证件 | 假学历、推荐信 |

## 3. 事件物品槽位匹配示例

### 示例1：送外卖事件
```yaml
事件ID: act3_work_delivery
物品槽位:
  - 槽位ID: transport_slot
    标签要求: [transport]        # 需要交通工具
    强制要求: true               # 必须有，否则无法执行
    默认选择: 优先级最高

槽位效果叠加:
  基础收入: 60
  基础行动点消耗: 2
  道具效果:
    - vehicle_tesla: { 收入倍率: 1.3, 行动点: -1 }
    - vehicle_scooter: { 收入倍率: 1.0, 行动点: -1 }
    - vehicle_ebike: { 收入倍率: 0.9, 行动点: 0 }
```

### 示例2：跨境向导事件
```yaml
事件ID: act2_move_guide
物品槽位:
  - 槽位ID: guide_slot
    标签要求: [guide]            # 需要向导
    强制要求: false              # 可选，有向导更容易
    默认选择: 优先级最高

槽位效果叠加:
  基础成功率: 50%
  道具效果:
    - smuggler_amir: { 成功率: +30%, 行动点: -1 }
    - guide_juan: { 成功率: +20% }
```

### 示例3：复合槽位事件（Uber司机）
```yaml
事件ID: act3_work_uber
物品槽位:
  - 槽位ID: transport_slot
    标签要求: [transport]
    强制要求: true
  - 槽位ID: identity_slot
    标签要求: [identity]         # 需要身份凭证
    强制要求: true

槽位效果叠加:
  基础收入: 120
  基础行动点消耗: 3
  两个槽位都必须满足才能执行事件
```
