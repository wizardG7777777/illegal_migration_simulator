# 终结态事件详细设计

> 本文档定义角色进入终结态后可触发的救援/应对事件。
> 基于 CharacterSystemArchitecture.md 中的终结态机制设计。

---

## 1. 概述

当角色资源归零时，会进入三种终结态之一：
- **濒死状态（DYING）**：身体健康度 ≤ 0
- **崩溃状态（BREAKDOWN）**：心理健康度 ≤ 0
- **匮乏状态（DESTITUTION）**：金钱 ≤ 0

本事件设计覆盖所有三个场景的终结态应对机制，每个场景有特定的救援事件池。

#### term_emergency_painkiller（服用止痛药）

| 字段 | 内容 |
|-----|------|
| **ID** | `term_emergency_painkiller` |
| **分类** | TERMINAL（终结态专用） |
| **适用场景** | 全场景 |
| **触发条件** | 处于【濒死】终结态，持有 `consumable_painkiller` |

**你看到了什么**：
> 你的视线开始模糊，伤口的剧痛让你几乎昏厥。颤抖的手摸向口袋——还有那瓶止痛药。

**可选行动**：

| 选项 | 条件 | 结果效果 |
|-----|------|---------|
| 吞下止痛药 | 持有止痛药×1 | 健康度+15，心理状态+5，脱离濒死状态；消耗止痛药×1 |
| 硬撑过去 | 无 | 50%几率进入下一回合濒死，50%几率立即死亡 |

## 6. 设计注意事项

1. **濒死紧迫性**：濒死状态的倒计时最短（0-3回合），因此救援事件效果应当显著（通常恢复20+健康度）。

2. **崩溃多样性**：崩溃状态应对手段较多，应鼓励玩家建立社交关系或储备药物。

3. **匮乏慢性化**：匮乏状态无倒计时但持续扣血，求生事件应该帮助玩家获得稳定收入来源而非一次性救济。

4. **同伴价值**：有同伴时触发同伴救援事件的概率显著增加，体现"关系网"的重要性。

5. **场景差异**：
   - 场景1：亲友、同乡会资源较多
   - 场景2：同伴依赖度高
   - 场景3：慈善机构、社区资源可用
